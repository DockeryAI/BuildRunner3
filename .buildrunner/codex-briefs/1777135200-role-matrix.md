## Claude's Current Hypothesis & Decisions (10 calls)

**1. Sandbox: writable-roots allowlist, not full bypass and not file restructuring.**
Keep codex in `workspace-write`. Don't use `--dangerously-bypass-approvals-and-sandbox` (one-way door). Don't restructure BR3's home-dir paths into the project root (those locations are correct for cluster-wide infra). Instead, add a per-project codex config (e.g. `.buildrunner/codex-sandbox.toml`) declaring additional writable roots, and have `runtime-dispatch.sh` append `-c sandbox_workspace_write.writable_roots=[...]` per invocation. BR3 ships with `~/.buildrunner/` and `~/Library/LaunchAgents/` allowlisted. Other projects ship no allowlist file and get standard project-root sandbox.

**2. Rule 22: rewrite, don't remove.** Change rule to "if file targets are outside project root AND outside declared writable_roots, demote to claude." Keeps the safety check working for projects that haven't opted in.

**3. Project runtime default: codex via runtime.json.** Flip BR3's `default_runtime` to `codex`.

**4. Global default for future projects: codex via bootstrap template, NOT env var.** A `BR3_DEFAULT_RUNTIME=codex` env var is invisible long-term. Change the project-init skill template so new `runtime.json` files default to codex automatically.

**5. `/spec-codex` writes runtime.json automatically.** Spec and runtime should never disagree. When `/spec-codex` runs, it sets `default_runtime: codex`.

**6. Inline dispatch is banned for autopilot phases.** Inline bypasses dispatch prefix, model/effort knobs, AND the post-phase review hook. Make autopilot phase hook the only path.

**7. Per-phase review: Sonnet reviewer, Opus 4.7 arbiter on disagreement only.** Loop: codex executes → Sonnet reads diff vs Success Criteria → approve/reject → on reject, codex gets one revision → if Sonnet still rejects, Opus 4.7 arbitrates with both sides. Reuse `cross_model_review.py`; change trigger from plan-time to phase-time.

**8. One revision per phase, then arbiter — no infinite loops.** Apply fixit skill's "hard cap one review pass per dispatch" rule to per-phase loop.

**9. Stale codex alert: fix the freshness check at the source.** `check-runtime-alerts.sh` should treat missing/empty state files as "no signal" not "last error stands forever."

**10. Validate on a clean project before re-running BUILD_burnin-queue-v2.** Don't make the worst-case build (entirely home-dir paths) the first test of the new flow.
